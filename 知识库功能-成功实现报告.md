## 🔧 技术要点总结

### H2数据库列名引用规则
1. **表名必须引用：** `"knowledge_article"`
2. **列名在DML中引用：** `"title"`, `"article_type"` 等
3. **列名在WHERE中引用：** `"deleted" = 0`
4. **列名在ORDER BY中引用：** `ORDER BY "created_at" DESC`

### MyBatis注解
```java
@Mapper
public interface KnowledgeArticleMapper {
    @Insert("INSERT INTO \"table\" (\"col1\", \"col2\") VALUES (#{val1}, #{val2})")
    @Options(useGeneratedKeys = true, keyProperty = "id")
    int insert(Entity entity);

    @Select("SELECT * FROM \"table\" WHERE \"col\" = #{val}")
    List<Entity> select(@Param("val") String val);
}
```

## 📊 经验教训

### 失败15次的原因
1. Hibernate ORM在H2文件数据库的表名映射问题
2. data.sql只执行一次，无法重新初始化表
3. 大小写敏感性的配置冲突

### 成功的关键
1. **采用完全不同的技术方案** - MyBatis vs Hibernate
2. **手动创建表** - 绕过data.sql限制
3. **精确控制SQL** - 所有标识符都使用双引号
4. **逐步验证** - 先创建表，再测试SELECT，最后测试INSERT
