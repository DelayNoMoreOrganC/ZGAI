### 3. 优化收款率统计 (getCollectionRate)

**之前**: 固定的假数据

**优化后**: 真实的应收/实收计算
```java
// 应收款（从案件的attorneyFee字段统计）
BigDecimal monthReceivable = monthCases.stream()
    .map(Case::getAttorneyFee)
    .filter(Objects::nonNull)
    .reduce(BigDecimal.ZERO, BigDecimal::add);

// 实收（从Payment表统计）
List<Payment> monthPayments = paymentRepository
    .findByPaymentDateBetween(monthStart, monthEnd);
BigDecimal monthReceived = monthPayments.stream()
    .map(Payment::getPaymentAmount)
    .filter(Objects::nonNull)
    .reduce(BigDecimal.ZERO, BigDecimal::add);

// 收款率
double rate = monthReceivable.compareTo(BigDecimal.ZERO) > 0
    ? monthReceived.divide(monthReceivable, 4, RoundingMode.HALF_UP)
        .multiply(new BigDecimal("100")).doubleValue()
    : 0;
```

**效果**:
- ✅ 真实的应收款统计
- ✅ 真实的实收款统计
- ✅ 准确的收款率计算（4位小数精度）
- ✅ 避免除零错误
