## 🚀 快速开始

### 访问系统

```
前端地址: http://localhost:3017
后端地址: http://localhost:8080/api
默认账号: admin / admin123
```

### 第一步：登录

1. 打开浏览器访问：http://localhost:3017
2. 输入账号：admin
3. 输入密码：admin123
4. 点击"登录"按钮

### 第二步：查看工作台

登录后自动进入工作台，您将看到：
- **统计卡片**：本月案件数、进行中案件、本月开庭、待办数、本月收费
- **日历视图**：显示开庭、审限等日程安排
- **待办事项**：按紧急程度排序的任务列表
- **AI智能创建**：上传法律文书，AI自动创建待办/日程

## 📋 核心功能使用指南

### 1. 案件管理

#### 1.1 查看案件列表
- **路径**：登录后 → 点击左侧菜单"案件管理" → "案件列表"
- **功能**：
  - 筛选：按类型/状态/等级/律师/法院/时间筛选
  - 排序：点击列名排序
  - 视图切换：列表视图 ↔ 看板视图
  - 批量操作：选中多个案件后进行批量操作

#### 1.2 新建案件
- **路径**：点击"新建案件"按钮
- **步骤**：
  1. 填写A-F六个部分表单
     - A. 基本信息（案件类型、名称、案号等）
     - B. 当事人（点击"添加当事人"）
     - C. 代理律师费
     - D. 应收款（点击"添加应收款"）
     - E. 结案/归档信息（如适用）
     - F. 关联信息（关联客户、关联案件）
  2. 可选：点击"AI智能填充"上传法律文书自动识别
  3. 可选：点击"查重"检查是否有重复案件
  4. 点击"提交"创建案件

#### 1.3 查看案件详情
- **路径**：在案件列表点击案件名称
- **5个Tab页**：
  - **Tab① 基本案情**：案件信息、当事人、费用
  - **Tab② 办案记录**：添加记录、导出Word/Excel
  - **Tab③ 受理单位**：保全、执行、庭审、承办人
  - **Tab④ 案件文档**：卷宗管理、AI识别、在线预览
  - **Tab⑤ 案件动态**：操作日志、评论、@提及

#### 1.4 编辑案件
- **路径**：在案件详情页点击"编辑"按钮
- **步骤**：修改信息后点击"保存"

#### 1.5 批量操作
- **路径**：在案件列表勾选多个案件 → 点击"批量操作"
- **可选操作**：
  - 批量结案
  - 批量归档
  - 修改主办律师
  - 批量删除

### 2. AI智能辅助

#### 2.1 AI OCR智能识别
- **位置**：案件详情 → 案件文档Tab → 点击"AI识别"
- **功能**：上传法律文书图片/PDF，自动提取案件信息
- **支持**：起诉状、判决书、仲裁书等
- **流程**：上传 → OCR识别 → LLM提取 → 人工校验 → 确认回填

#### 2.2 AI文书生成
- **位置**：案件详情页 → 点击"AI文书生成"
- **可选类型**：
  - 起诉状初稿
  - 答辩状初稿
  - 代理词框架
  - 法律意见书框架
- **步骤**：
  1. 选择文书类型
  2. 点击"生成"
  3. 等待AI生成（约3-5秒）
  4. 编辑后使用

#### 2.3 AI法律知识问答
- **位置**：知识库 → AI知识问答
- **使用方法**：
  1. 输入法律问题（如："民事诉讼时效是多久？"）
  2. 点击"提问"
  3. 系统基于知识库+AI生成答案
  4. 显示相关法律依据

**技术说明**：
- 当前使用：Ollama本地AI（qwen3:8b模型）
- 检索方式：TF-IDF关键词检索
- 响应时间：约3秒

## ❓ 常见问题

### Q1: 登录后看不到数据？
**A**: 检查是否正确登录。如果还是看不到，可能是数据为空，先创建一个测试案件。

### Q2: 点击"编辑"按钮没反应？
**A**: 这是已修复的功能。刷新页面重试。如果还不行，检查浏览器控制台是否有错误。

### Q3: AI问答没有响应？
**A**: 检查Ollama是否运行：
```bash
curl http://localhost:11434/api/tags
```
如果没有返回模型列表，需要启动Ollama。

### Q4: 批量操作不工作？
**A**:
1. 确保已选中案件（勾选复选框）
2. "批量操作"按钮才会显示
3. 选择操作后按提示完成

### Q5: 系统链接点击无效？
**A**: 所有链接已修复（路由完整性100%）。如果还是不工作，请清除浏览器缓存后重试。

### Q6: 工作台数据不显示？
**A**:
1. 检查后端服务是否运行（http://localhost:8080）
2. 检查前端控制台是否有错误
3. 刷新页面重试
