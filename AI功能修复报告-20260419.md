## 根因分析

### 问题链路

1. **JWT Token生成** (`JwtUtil.java:35-39`):
```java
public String generateToken(Long userId, String username) {
    Map<String, Object> claims = new HashMap<>();
    claims.put("userId", userId);      // userId = 1
    claims.put("username", username);  // username = "admin"
    return generateToken(claims);
}
```

2. **JWT Filter解析** (`JwtAuthenticationFilter.java:59-60`):
```java
UsernamePasswordAuthenticationToken authentication =
    new UsernamePasswordAuthenticationToken(userId, null, authorities);
//                                         ^^^^^^
//                                         问题：将userId作为principal
```

3. **Controller获取用户** (原代码):
```java
private Long getCurrentUserId() {
    String username = SecurityContextHolder.getContext()
                          .getAuthentication()
                          .getName();  // 返回 "1"（userId），而不是 "admin"
    return userRepository.findByUsername(username)
            .map(User::getId)
            .orElseThrow(() -> new RuntimeException("用户不存在: " + username));
}
```

### 执行流程

```
JWT生成: userId=1, username="admin"
    ↓
Filter设置: principal=1 (Long类型)
    ↓
Controller: getName() 返回 "1"
    ↓
查询: userRepository.findByUsername("1")
    ↓
结果: 找不到用户 → 抛出异常
```

## 解决方案

### 1. 创建通用工具类

**文件**: `D:\ZGAI\backend\src\main\java\com\lawfirm\util\SecurityUtil.java`

```java
@Component
@RequiredArgsConstructor
public class SecurityUtil {

    private final UserRepository userRepository;

    /**
     * 获取当前用户ID - 兼容两种principal类型
     */
    public Long getCurrentUserId() {
        Object principal = SecurityContextHolder.getContext()
                                              .getAuthentication()
                                              .getPrincipal();

        // 兼容Long类型（userId）
        if (principal instanceof Long) {
            return (Long) principal;
        }

        // 兼容String类型（username）
        String username = principal.toString();
        return userRepository.findByUsername(username)
                .map(User::getId)
                .orElseThrow(() -> new RuntimeException("用户不存在: " + username));
    }

    public String getCurrentUsername() { /* ... */ }

    public User getCurrentUser() { /* ... */ }
}
```

### 2. 修复DocGenerateController

**修改前**:

## 长期解决方案

### 方案A: 修改JwtAuthenticationFilter（推荐）

修改Filter，将username作为principal：

```java
// JwtAuthenticationFilter.java:60
UsernamePasswordAuthenticationToken authentication =
    new UsernamePasswordAuthenticationToken(username, null, authorities);
//                                         ^^^^^^^^
//                                         改为使用username
```

**优点**:
- 符合Spring Security惯例（principal通常是username）
- 不需要修改现有Controller

**缺点**:
- 需要全面测试（影响所有Controller）

### 方案B: 保持现状 + SecurityUtil

保持Filter不变，在需要的Controller中使用SecurityUtil。

**优点**:
- 不影响现有代码
- 渐进式修复

**缺点**:
- 不符合Spring Security惯例
- 需要在多处添加工具类调用
